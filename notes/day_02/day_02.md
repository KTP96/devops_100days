## Day-2 | Improve SDLC with DevOps

### SDLC (Software Development Life Cycle)

SDLC is the **step-by-step process** used to design, develop, test, deploy, and maintain software.

It helps teams build software in a **planned, organized, and reliable** way.

### SDLC Stages

1. **Planning**
   - define project scope
   - estimate time, cost, and resources
   - identify tools, team, and approach

2. **Define**
   - gather and document requirements
   - understand business and user needs
   - create requirement specifications

3. **Design**
   - prepare system architecture and technical design
   - create **HLD (High-Level Design)** and **LLD (Low-Level Design)**
   - decide modules, workflows, database, and interfaces

4. **Build**
   - developers write the code
   - DevOps may support build tools, version control, and CI setup
   - application is prepared for testing

5. **Test**
   - verify the application works correctly
   - find bugs and validate requirements
   - DevOps helps automate tests and manage test environments

6. **Deploy**
   - release the application to staging or production
   - DevOps handles CI/CD pipelines, automation, configuration, and release process

### DevOps-Focused Stages

DevOps is mainly focused on:

- **Build**
- **Test**
- **Deploy**

DevOps can also support **Planning** and **Design** for:

- infrastructure planning
- CI/CD design
- environment architecture
- deployment strategy
