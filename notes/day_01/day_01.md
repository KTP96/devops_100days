## Day 1 | Fundamentals of DevOps

### What is DevOps?

DevOps is a **culture and set of practices** that improves software delivery by combining:

- development
- operations
- automation
- testing
- monitoring

Its goal is to deliver software **faster, reliably and efficiently**.

### Why DevOps?

DevOps is used to:

- speed up software delivery
- improve software quality
- reduce manual effort
- increase system reliability
- improve teamwork between developers and operations teams

### Day-to-Day Activities of a DevOps Engineer

Common day-to-day activities include:

- building and maintaining **CI/CD pipelines**
- automating repetitive manual tasks
- managing **cloud infrastructure**
- monitoring applications and servers
- troubleshooting build, deployment, and production issues
- improving **system reliability, scalability, and security**
- working with developers, QA, and operations teams
- managing containers and orchestration tools like **Docker** and **Kubernetes**
- handling **infrastructure as code** tools like **Terraform**
- setting up logging, alerting, and observability tools
- supporting release processes and environment management
